# Documentation Writer Skill

Technical documentation creation skill for writing module specifications, API contracts, architecture decision records (ADRs), and feature specifications. Follows the 6 C's quality framework.

## When to Use

- Creating technical documentation for a module
- Writing API contract documentation
- Documenting architecture decisions (ADR)
- Creating feature specifications

## Quick Reference

### Document Types

| Type | Purpose |
|------|---------|
| Module Specification | Technical spec for implementation |
| API Contract | Endpoint documentation |
| ADR | Architecture decisions |
| Feature Specification | User stories and requirements |

### Output Locations

| Document Type | Directory |
|---------------|-----------|
| Module Specs | `/docs/module-specs/` |
| API Contracts | `/docs/api-contracts/` |
| ADRs | `/docs/adrs/` |
| User Requirements | `/docs_dev/requirements/` |
| Process Docs | `/docs/workflows/` |

### Quality Framework (6 C's)

- **Complete** - all aspects covered
- **Correct** - technically accurate
- **Clear** - unambiguous language
- **Consistent** - same terminology throughout
- **Current** - reflects latest decisions
- **Connected** - cross-references to related docs
